API REST

Desarrollado en Spring Boot con una arquitectura monolítica empleando el patrón de diseño Modelo-Vista-Controlador (MVC). Se implementaron Json Web Token (JWT) para la autenticación, y encriptación para las credenciales y así garantizar seguridad a los usuarios. Además, se utilizó una metodología Domain Driven Design (DDD) la cual permite modularizar las entidades, controladores y servicios de la aplicación.

Ver código

Esta estructura permite tener las responsabilidades separadas y permite escalar la aplicaciónde forma eficiente. Algunos patrones aplicados:

  • CQRS (Command Query Responsability Segregation)
  • ACL (Anti-Corruption Layer)
  • Repository
  • DTO (Data Transfer Object)

Microservicio que permite alojar configuraciones, dependencias, credenciales, etc. de otros microservicios.

Eureka
Microservicio que permite gestionar a sus clientes (otros microservicios)

Gateway
Microservicio por el cual se realizan las peticiones y este se encargará de redirigir al que corresponde
